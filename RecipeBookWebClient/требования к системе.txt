Требования к ПО «Книга рецептов»

Нефункциональные требования
Система должна быть реализована в виде веб‑приложения или мобильного приложения‑клиента с отдельной серверной частью (т.е. бэкэнд обязательно должен быть в любом случае).
Система должна хранить данные о продуктах и блюдах в базе данных или в файлах на стороне сервера.
Серверная часть должна предоставлять API для взаимодействия с ней.
Код должен обеспечивать возможность написания тестов: юнит‑тестов, интеграционных или API тестов, UI тестов.


Функциональные требования
1. Управление продуктами
1.1 Создание продукта
Пользователь должен иметь возможность создавать новые продукты с указанием названия, фотографий, калорийности, БЖУ на 100 грамм, состава, категории, необходимости готовки и дополнительных флагов. Сумма БЖУ на 100 грамм не может превышать 100.
1.2 Отображение продуктов
Система должна отображать список всех продуктов с возможностью:

Фильтрации по категории, необходимости готовки и дополнительным флагам (веган, без глютена, без сахара). Фильтры могут комбинироваться.
Поиска по названию продукта (поиск по подстроке, не регистрозависимый).
Сортировки по следующим критериям: название (в алфавитном порядке), калорийность, белки, жиры, углеводы.
1.3 Просмотр продукта
Пользователь должен иметь возможность просмотреть выбранный продукт со всеми его атрибутами.
1.4 Редактирование продукта
Пользователь должен иметь возможность редактировать существующие продукты, включая изменение любых доступных для редактирования атрибутов. Сумма БЖУ на 100 грамм не может превышать 100.
1.5 Удаление продукта
Пользователь должен иметь возможность удалять продукты, которые больше не нужны. ВНИМАНИЕ! Удаление продукта, который используется в составе хотя бы одного блюда, должно быть недоступно. Система должна информировать пользователя о невозможности удаления и указывать блюда, в которых используется данный продукт.


2. Управление блюдами
2.1 Создание блюда
Пользователь должен иметь возможность создавать новые блюда с указанием названия, фотографий, калорийности, БЖУ на порцию, состава (набор продуктов с указанием количества каждого продукта в порции), размера порции, категории и дополнительных флагов. Сумма БЖУ на 100 грамм не может превышать 100.
2.2 Автоматический расчёт КБЖУ
При создании и редактировании блюда система должна автоматически рассчитывать калорийность, белки, жиры и углеводы на порцию исходя из состава блюда (на основе КБЖУ входящих продуктов на 100 г и указанного количества каждого продукта). Рассчитанные значения подставляются в форму в качестве черновых значений, которые пользователь может скорректировать при необходимости.
2.3 Автоматическое определение категории блюда (макросы)
Если в названии блюда указан один из макросов !десерт, !первое, !второе, !напиток, !салат, !суп, !перекус, система должна автоматически присвоить блюду соответствующую категорию, при этом макрос будет удалён из названия:

!десерт → категория «Десерт»
!первое → категория «Первое»
!второе → категория «Второе»
!напиток → категория «Напиток»
!салат → категория «Салат»
!суп → категория «Суп»
!перекус → категория «Перекус»

Если макросов указано несколько, применяется только первый указанный, остальные игнорируются.
ВНИМАНИЕ! Данная возможность НЕ отменяет возможность задания категории в соответствующем поле формы создания блюда. При задании категории одновременно в названии и в поле формы будет установлено значение из поля формы.
2.4 Управление дополнительными флагами блюда
Дополнительные флаги блюда (веган, без глютена, без сахара) доступны для установки только при соблюдении следующих условий:

Флаг «Веган» — доступен для установки только если все продукты в составе блюда отмечены флагом «Веган».
Флаг «Без глютена» — доступен для установки только если все продукты в составе блюда отмечены флагом «Без глютена».
Флаг «Без сахара» — доступен для установки только если все продукты в составе блюда отмечены флагом «Без сахара».

Если хотя бы один продукт в составе не имеет соответствующего флага, установка данного флага у блюда должна быть недоступна.

При изменении состава блюда (добавление/удаление продукта) доступность флагов должна пересчитываться. Если ранее установленный флаг более не может быть применён, он должен быть автоматически снят.
2.5 Отображение блюд
Система должна отображать список всех блюд с возможностью:

Фильтрации по категории блюда (Десерт, Первое, Второе, Напиток, Салат, Суп, Перекус) и дополнительным флагам (веган, без глютена, без сахара). Фильтры могут комбинироваться.
Поиска по названию блюда (поиск по подстроке, не регистрозависимый).
2.6 Просмотр блюда
Пользователь должен иметь возможность просмотреть выбранное блюдо со всеми его атрибутами, включая полный состав (перечень продуктов с количеством).
2.7 Редактирование блюда
Пользователь должен иметь возможность редактировать существующие блюда, включая изменение любых доступных для редактирования атрибутов. При изменении состава блюда КБЖУ автоматически пересчитывается (с сохранением возможности ручной корректировки пользователем). Сумма БЖУ на 100 грамм не может превышать 100.
2.8 Удаление блюда
Пользователь должен иметь возможность удалять блюда, которые больше не нужны.

Правила автоматического расчёта КБЖУ блюда
При создании или редактировании блюда система автоматически рассчитывает КБЖУ на порцию по следующим формулам:

Калорийность порции = Σ (калорийность продукта на 100 г × количество продукта в порции / 100) — для всех продуктов в составе
Белки порции = Σ (белки продукта на 100 г × количество продукта в порции / 100) — для всех продуктов в составе
Жиры порции = Σ (жиры продукта на 100 г × количество продукта в порции / 100) — для всех продуктов в составе
Углеводы порции = Σ (углеводы продукта на 100 г × количество продукта в порции / 100) — для всех продуктов в составе

Рассчитанные значения подставляются в форму как черновые. Пользователь может скорректировать любое из этих значений вручную.
